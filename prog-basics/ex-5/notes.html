Level 1

Thought Process

The objective of this level was to install the dotenv library and use it to load configuration variables from a .env file. The variables, such as PORT and DB_HOST, were created in the .env file and loaded into the program using dotenv.config(). The process of reading and using these values for program configuration was straightforward, given my prior experience with environment variables in training projects.

Level 2

Thought Process

In this level, I enhanced the program to include fallback values for missing environment variables. Additionally, I added functionality to log warnings when a variable was missing. This ensures the program handles missing variables gracefully without failing. Implementing these features involved checking for undefined variables and providing defaults where necessary, which was straightforward due to my familiarity with handling environment variables.

Level 3

Thought Process

The focus of this level was to use environment variables to configure a database connection and API credentials. For this exercise, I used a SQLite file-based database. I ensured that sensitive configuration details were logged securely, avoiding exposure of critical information such as API keys. Implementing secure logging and configuring the database connection was straightforward due to my existing knowledge and prior experience with similar setups.
