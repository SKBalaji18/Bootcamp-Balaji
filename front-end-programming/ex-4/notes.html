Vite Full Exercise - Exercise Notes

Process

This exercise focused on using Vite to its full potential, implementing advanced features and best practices. The first task involved setting up environment-specific configurations by creating .env.development and .env.production files to define separate API endpoints for development and production environments. The app fetches data from the appropriate endpoint based on the environment.

Advanced Vite practices were applied by integrating vite-plugin-checker for real-time TypeScript type-checking and vite-plugin-eslint for linting during development. Errors are displayed during the build process.

The final task involved building upon the previous work from Exercise 3 (TypeScript version) by adding advanced features like lazy loading, code splitting, and production optimizations such as minification and tree-shaking. Performance testing was also conducted on the final build.

Challenges
